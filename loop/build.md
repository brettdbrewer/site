# Build Report — Invite Teammates Card

## Gap
After completing the Getting Started checklist in a new solo space (member_count < 2), there was no prompt to invite collaborators. The space had just proven its value (task created, agent task created, first completion) but left the user alone.

## Changes

### `graph/handlers.go`
- In `handleBoard`: after welcome modal logic, query `h.store.MemberCount` when the checklist is complete (`showChecklist && hasTask && hasAgentTask && hasCompletion`).
- If `memberCount < 2`, set `showInviteCard = true` and get/create an invite token via `GetInviteToken`/`CreateInvite`.
- Pass `showInviteCard` and `inviteToken` to `BoardView`.

### `graph/views.templ`
- Updated `BoardView` signature: added `showInviteCard bool, inviteToken string`.
- Added `if showInviteCard { @inviteTeammatesCard(...) }` render call after `celebrationCeremony`.
- Added new `inviteTeammatesCard(spaceSlug, inviteToken string)` component:
  - Inline card (not modal) above the board columns.
  - Shows user icon, heading "Start collaborating", message "Invite your team to build together with your AI peer."
  - If token exists: shows read-only invite link input + "Copy" button with clipboard JS.
  - If no token: shows "Generate invite link" form POST.
  - Dismiss button removes card and sets `localStorage` key to suppress on subsequent visits.

### `graph/views_templ.go`
- Regenerated by `templ generate`.
